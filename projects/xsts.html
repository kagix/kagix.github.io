XSTS

XSTS is about improving the user ride experience in public transportation. Many city councils have little to no budget for developping better transit service. What could have been considered true 50 years ago can and should be heavily challenged in the age of digital information.

This is not a project but a framework made of many layers and bricks, generating information that can make rider's life better. Many images and all the PDF documents from my blog have been created automatically by one of the many modules of XSTS.

XSTS is written in Java and you can download the code for free from the official repository.

The framework works only with datasets containing a single agency. The following modules are part of XSTS and you can download them individually from the repository above.

The top level projects

- The Core module used by all the other ones.
- The PDF module is responsible of creating tho compact schematics and booklets files you might have seen on my blog. It is GTFS agnostic and you can use it to create your own PDF files.
- The Tutorial module is probably the most important for you. It shows how to use the XSTS modules in order to profit from the framework.

The GTFS-related projects

- The GTFS Data Structure module implements a simple object model (POJO) for the respective files from the GTFS datasets. It supports the required subset of files and a some of the optional ones.
- The GTFS Data Load module is responsoble from reading the content of the datasets from either plain text, or zip files. It is very easy to adapt to a SQL/NoSQL database and most likely I will add such capabilities in the future.
- The GTFS Splitter module is responsible of breaking large module used by all the other ones. This is the component which breaks down large GTFS datasets and creates individual single agency GTFS files. I have tested it successfully on very large (50-100MB) multy-agency files, like the Swiss national GTFS, which contains literally hundreds of agencies.

The Printable Transit Information group

Note: these projects are not yet public. In case you wonder how I created the dozens and dozens of schematics you can find on the blog site, the answer is below.
- The Stop Trek Voyager module takes a GTFS dataset and creates a PDF file with the schematics for a line, a group of lines or the whole network. I have tested it with Latin, Cyrillic, Chinese and Japanese characters. For each schematics you can have the relative time between a specific stop and all the other for that route, as well as the distance between stations and the connections for each stop.
- The Transit Network Booklet is based on the great pocket guides from the 70s and 80s that an agency used to print every 5 to 10 years. The last hardcopy that I am aware of dates back to the 90s. This standalone app takes a GTFS dataset and creates a pocket guide with pages for the individual lines. For each stop you have the list of connections.
